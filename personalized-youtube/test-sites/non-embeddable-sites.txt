# Non-embeddable sites — fall back to ARCHETYPE reconstruction
#
# Big, reputable sites that cannot be shown as a live iframe. The hybrid
# renderer detects this during ingestion and rebuilds the page from its
# detected frontend profile (mode/accent/font/layout) + content, with an
# honest note.
#
# Reason codes:
#   (H) HEADER block  — sends X-Frame-Options or CSP frame-ancestors
#   (J) JS frame-bust — permissive headers, but busts/blanks itself in a frame
#   (L) Login wall    — content stays behind sign-in even after you log in
#
# Verified May 2026.

# --- Big consumer sites (header-blocked) ---
https://www.youtube.com              # (H) XFO: SAMEORIGIN
https://www.google.com               # (H) XFO: SAMEORIGIN
https://www.reddit.com               # (H) XFO: SAMEORIGIN
https://www.nytimes.com              # (H) XFO: DENY
https://www.bbc.com                  # (H) XFO: SAMEORIGIN
https://github.com                   # (H) XFO: deny + CSP frame-ancestors 'none'
https://stripe.com                   # (H) XFO: SAMEORIGIN + CSP frame-ancestors

# --- Developer / foundation sites (header-blocked) ---

https://www.apache.org               # (H) CSP frame-ancestors 'none'
https://www.gnu.org                  # (H) XFO: sameorigin
https://www.ietf.org                 # (H) XFO: DENY
